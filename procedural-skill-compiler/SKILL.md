# Procedural Skill Compiler

把重复任务编译成"肌肉记忆"—— Hermes 学会后只需一句话就能触发。

## 编译流程

### 第一步：识别重复模式

当同一类任务出现 ≥2 次时，触发编译：

```
识别信号：
- 用户多次让你执行相同结构的复杂任务
- 工作流涉及 3+ 工具调用
- 任务有明确的输入→处理→输出模式
```

### 第二步：提炼 Skill 骨架

从对话中提取：
1. **触发词**（用户说什么时调用）
2. **输入**（什么数据/文件/参数）
3. **步骤**（哪几步，工具调用顺序）
4. **输出**（发给谁，投递到哪里）

### 第三步：写入 Skill 文件

路径：`~/.hermes/skills/<skill-name>/SKILL.md`

格式：
```markdown
---
name: <skill-name>
description: <触发词+一句话说明>
---

# <Skill 名称>

## 触发词
「<触发短语>」

## 输入
- 

## 步骤
1. 

## 输出
- 

## 示例
「<示例输入>」→「<预期输出>」
```

### 第四步：通知用户

编译完成后告知：
```
📦 肌肉记忆已编译：「<skill-name>」
以后只需说「<触发词>」，我就会自动执行以下步骤...
```

## 编译触发条件

| 条件 | 阈值 |
|------|------|
| 同一任务出现次数 | ≥ 2次 |
| 工具调用数 | ≥ 3步 |
| 用户明确要求"记住" | 立即编译 |

## 不编译的情况

- 一次性分析、创意任务
- 涉及外部 API key 的敏感操作
- 需要用户每次确认的操作
