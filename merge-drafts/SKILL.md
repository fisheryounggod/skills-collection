# 多稿合并技能

## 触发方式

以下任一表达均可触发此技能：
- 合并稿子
- 合稿
- merge drafts
- 把这几篇合成一篇
- 综合这几份稿子
- 合并文章

## 写作风格

本技能遵循一致的写作风格规范，确保合并后的文章读起来像一个人写的。

## 输入要求

- 接收 2 份或以上草稿文件路径
- 同时读取同目录下的 analysis.md（如有）作为素材参考
- 读取当前目录下的相关配置文件（如有）

## 工作流程

### 步骤一：阅读所有稿子

快速评估每份草稿的：
- 结构完整性
- 信息覆盖面
- 表达质量
- 独特亮点

### 步骤二：选择基础稿

标准（按优先级排序）：
1. 结构最清晰
2. 信息最全面
3. 表达质量最好
4. 改动最小原则

选出最佳稿作为合并的基础。

### 步骤三：提取各稿精华

逐一审阅其他稿子，提取：
- 缺失内容
- 更好的表达方式
- 独特角度
- 有价值的数据和案例

### 步骤四：融合合并

在基础稿上进行合并操作：
- 补充缺失内容
- 替换更优表达
- 融合不同视角
- 统一风格

**核心原则：融合而非拼接，合并后读起来像一个人写的。**

### 步骤五：润色检查

- 消除拼接痕迹
- 确保风格统一
- 删除重复内容
- 检查逻辑连贯性
- 优化段落过渡

### 步骤六：输出合并报告

报告内容包括：
- 基础稿选择理由
- 各稿贡献内容
- 主要修改说明

## 输出格式

合并完成后：
1. 输出合并后的完整文章
2. 提供合并报告说明修改内容
