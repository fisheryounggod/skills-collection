# Open Slide

Open or navigate an HTML slide deck, especially decks generated by `frontend-slides`, `html-ppt`, or other local presentation skills.

## When To Use

Use this skill when the user asks to:

- `使用 open-slide`
- open a generated HTML presentation
- jump to a specific slide number
- navigate the current in-app browser slide deck
- verify the currently visible slide

## Workflow

1. Identify the deck URL or file path.
   - If the prompt includes in-app browser context, use the current URL first.
   - If the user gives a local `.html` file, convert it to a `file://` URL.
   - If no target is given, use the most recent generated HTML deck in the current working directory.

2. Identify the slide number.
   - Prefer an explicit number from the user.
   - Otherwise preserve the current `#N` hash if present.
   - If neither is available, open slide 1.

3. Navigate using the safest available method.
   - For decks with hash routing, use `file:///path/to/deck.html#N`.
   - For decks that support query routing, use `?open-slide=N`.
   - For decks that expose JavaScript helpers, call `window.openSlide(N)` or `window["open-slide"](N)`.
   - If a browser automation or in-app browser tool is available, prefer it for the current tab.
   - Otherwise use the platform opener, for macOS: `open 'file:///path/to/deck.html#N'`.

4. Confirm the result briefly.
   - Report the opened file and slide number.
   - If navigation could not be verified, say so and give the URL used.

## HTML Deck Compatibility Snippet

When editing a local HTML deck and the user wants `open-slide` support, add a small public API to the presentation controller:

```js
window.openSlide = (slideNumber) => controller.openSlide(slideNumber);
window["open-slide"] = (slideNumber) => controller.openSlide(slideNumber);
window.addEventListener("open-slide", (event) => {
  controller.openSlide(event.detail?.slide ?? event.detail);
});
```

Also support initial slide routing:

```js
const params = new URLSearchParams(location.search);
const initialSlide = Number(params.get("open-slide")) || Number(location.hash.replace("#", ""));
if (initialSlide) controller.openSlide(initialSlide, "auto");
```

## Verification

When practical, verify:

- the browser URL has the expected `#N` or `?open-slide=N`
- the visible slide index matches the requested page
- keyboard navigation still works after opening

Keep the response short. This skill is operational, not a design review.
